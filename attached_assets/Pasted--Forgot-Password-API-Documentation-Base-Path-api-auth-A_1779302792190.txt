# Forgot Password API Documentation

Base Path: `/api/auth`

Authentication: No authentication is required for the forgot password flow. Users request a password reset OTP with their registered email, then use that OTP to reset their password.

---

## 1. Request Password Reset OTP

Sends a password reset OTP to the registered email address for the specified user.

**Endpoint:** `POST /forgot-password`

**Request Body:**
```json
{
  "email": "user@example.com"
}
```

**Validation:**
- `email` is required
- `email` must be a valid email format

**Behavior:**
- If the email belongs to a registered user, an OTP is generated and stored for that user.
- The OTP is sent asynchronously by email using the backend's password reset email service.
- If the email is not registered, the API returns a `user not found` error.

**Response (200 OK):**
```json
{
  "message": "Password reset OTP sent to your email"
}
```

**Error Response (400 Bad Request):**
```json
{
  "error": "user not found"
}
```

---

## 2. Reset Password

Resets the user's password using the email, OTP, and new password.

**Endpoint:** `POST /reset-password`

**Request Body:**
```json
{
  "email": "user@example.com",
  "otp": "123456",
  "new_password": "NewStrongPassword123!"
}
```

**Validation:**
- `email` is required and must be a valid email
- `otp` is required
- `new_password` is required and must be between 6 and 50 characters

**Behavior:**
- The backend checks that the user exists and that the provided OTP matches the stored OTP.
- If the OTP is valid, the new password is validated for strength, hashed, and saved.
- The stored OTP is cleared after the password is successfully reset.

**Response (200 OK):**
```json
{
  "message": "Password reset successfully"
}
```

**Error Responses (400 Bad Request):**
```json
{
  "error": "user not found"
}
```

```json
{
  "error": "invalid OTP"
}
```

```json
{
  "error": "password is not strong enough"
}
```

---

## Notes

- The OTP is generated by `Infrastructure.GenerateOTP()` and sent by `Infrastructure.SendPasswordResetOTP()`.
- This flow requires a correct, registered email address; the reset email is only sent for existing users.
- The backend currently uses OTP verification rather than a password reset link.
